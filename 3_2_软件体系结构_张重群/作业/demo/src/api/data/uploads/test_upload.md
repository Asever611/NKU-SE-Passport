# 软件体系结构基础

## 第一章 概述

软件体系结构（Software Architecture）是软件系统的高层结构描述，它定义了系统的组件、组件之间的关系以及指导其设计和演化的原则。

### 1.1 软件体系结构的定义

软件体系结构由构件（Component）、连接件（Connector）和约束（Constraint）三要素组成。

**构件**是可以是一组代码，如程序的模块，也可以是独立的程序，如数据库服务器。

**连接件**用于表示构件之间的相互关系，可以是过程调用、管道、消息传递等。

**约束**是组件连接时的条件，规定了构件如何组合在一起的规则。

### 1.2 软件体系结构风格

常见的体系结构风格包括：

1. **数据流风格**：包括批处理（Batch Sequential）和管道-过滤器（Pipe-Filter）。数据在处理器之间流动，数据的可用性控制着计算。

2. **调用/返回风格**：包括主程序-子程序、面向对象风格和层次结构风格。这是最广泛使用的风格。

3. **独立构件风格**：包括事件驱动和进程通信。构件之间通过消息传递进行松耦合通信。

4. **虚拟机风格**：包括解释器和基于规则的系统。提供了高度的灵活性和可移植性。

5. **数据为中心风格**：包括仓库系统和黑板系统。多个构件通过共享的数据存储进行通信。

### 1.3 4+1视图模型

Philippe Kruchten 提出的4+1视图模型包括：

- **逻辑视图**：描述系统的功能需求，包括主要的类和对象
- **开发视图**：描述软件模块的组织结构
- **进程视图**：描述系统的并发和同步机制
- **物理视图**：描述软件到硬件的映射关系
- **场景视图**（+1）：用一组用例场景将上述四个视图串联起来

## 第二章 智能体设计模式

### 2.1 路由模式（Routing）

路由模式使智能体系统能够根据环境状态、用户输入，在多个潜在动作之间动态决策。

### 2.2 RAG模式（Retrieval Augmented Generation）

RAG通过检索外部知识库来增强LLM的回答，减少幻觉，提供可追溯的答案来源。
