# NKU-SE-Passport
>✈️ 南开大学软件工程本科通行证

## 📂 仓库内容
* 📖 本仓库为本人在南开大学软件工程专业本科阶段学习的所有资料

* 📋 具体包括各科的`课件`、`实验要求`、`本人作业`及`期末复习资料`，其中`期末复习资料`包括`真题`、`笔记`等等。
* ❌ **不包含**`英语`、`思政系列`及`通识选修课`
* ⏰ 本仓库整理始于本人大三阶段（2026.2），~~我大三才开智~~，所以大三的内容较为丰富完整，大一大二的比较残破简陋。

## ⚠️ 注意
* 🔒 本仓库中内容仅限于学习交流使用，因包含老师的课件，请勿恶意传播篡改或用于商业用途，**不要搞得大家没有资料用！**

## 💡 一些建议
* 🚫 大一大二的学习中**不要依赖AI**！尤其**写作业**的时候，尤其`C++`、`数据结构`等基础课程，有Bug就自己慢慢调试，很庆幸我上大一的时候AI还没那么发达，不然最后啥也不会了。
* 📝 期末复习的时候如果时间不够就**优先做近年真题**和模拟题（如果有的话），时间充沛就再做早年真题，闲的没事干可以做天南情报站的题（~~情报站都是假题~~）。**当然前提是先复习完课程内容！** 另外各门课期末复习课的时候可以**录音**，自己回去复习的时候再慢慢反复听，也可以用飞书将录音自动转为文字并生成总结。
* 🤖 写作业时少用AI，但在学习时充分利用，尤其`老师讲的一般需要自学`且`课件做的一般杂乱无章`且`部分科目课件是英文`。
    * 🔥 **焚诀**：将这个文档转变为一份详细的教程，要包含文档中所有内容并使其连贯，对于文档中描述不清晰的地方要进行详细的展开讲解，对于文档中遗漏的部分要进行补充，对于基础的、重要的地方要进行实例演示讲解。
    * 将课件上传AI，然后输入焚诀，则可得到条理清晰详细完整的讲解。（建议一次只讲解一个课件）
    * 这个工作`豆包`足以胜任，免费，且单次输出足够长足够详细，还有`对话置顶`功能方便随时查看和期末复习。
* ❗一定要重视**“内容由AI生成，请仔细甄别”**这句话。血的教训：机器学习导论要考一些最简单的python代码，我当时问了DeepSeek一个最简单的问题：“`@`符号的用法是什么”，由于我此前从未学过python，于是对于AI给出的答案深信不疑（毕竟显然这是个十分简单的问题，AI再傻也不至于这都错吧），第二天考试也是非常有缘的就考到了这个语法，我也是非常顺利的做错了🙏

## 📌 Tips
对一些课程的个人建议
#### 1. 高数 📐
高数一定要投入更多的精力，10学分对绩点影响十分大，我就是高数没考好，后面专业课大部分都4.0以及一部分3.7（软院的老师给分都挺好），但也保研无望。
#### 2. 思政系列 📖
平时不用学，期末背两三天就完事了，给分全看老师，一定远离一个叫wsl的。
#### 3. 大语 📝
这个课给分都挺好的，老师人也都挺好的（推荐ymz），跟中学语文不是一个东西。期末会有一个叫**考勤题**的题，需要对平时上课的内容有所了解，每个老师的上课内容不同（区别较大），老师自己每年的内容也有变化（区别较小）。**焚诀**：我的资料中有伟大学长总结的较详细的ymz的笔记，平时上课多多的拍ppt，根据ppt查找到笔记中本学期所涉及的部分然后背一背即可。
#### 4. 体育 🏃
详情见资料库中的体育文件夹。
#### 5. 专业课 💻
软院的老教师们出卷子基本上年年都**变化不大**，所以做往年题非常重要，考试基本上都是`90%原题或同类型题`+`一些新题`。
1. 📍 **高级语言程序设计** c++上机考试开始之前还有很长时间（一般会让提前半个小时到场，时间充足），可以在此期间预写一些常用代码，如排序函数（冒泡、快排）和链表类等。
2. 📍 **Java** 上机考试时用的是机房的老eclipse，非常难用，可以参考[这篇文章](https://blog.csdn.net/qq_62291388/article/details/130300848?fromshare=blogdetail&sharetype=blogdetail&sharerId=130300848&sharerefer=PC&sharesource=Asever611&sharefrom=from_link)的第一条来设置一下自动补全。
3. 📍 **机器学习导论** 上机作业基本上一周一次，前期的尽量完全自己做，后期的比较复杂建议完全ai，但要一定要有印象（尤其是一些特别的作业），比如我那年的考试里就考到了上机作业的内容（考试题目是写出coze平台的一条任务流，那次作业也就是让人们模仿助教展示的流程搭建一条任务流），其实只要作业是自己做的就十分简单，但要是一点印象也没有考试就白给。对于python代码的考察，只会考最最简单的，numpy的矩阵乘什么的（第一次上机的内容）。
4. 📍 **并行** 赶上换老师了，毕业博士生第一年教课，这老师其实讲的还行，不死板，不是念ppt，但是考试给人干懵了，平时一直说什么考试很简单，肯定不会很难，结果我发现这小哥哥似乎不知道什么是简单什么是难。首先就是考试重点全变了，跟往年题重复率不到1%，唯一不变的是题型，所以必须仔仔细细的看他期末总结ppt的每一个角落。（我感觉是因为他过于重视上级`三年内不许出重复题`的要求了）
5. 📍 **数据库/操作系统** 恩师无需多言，给分最好没有之一，考试你就写吧，写上就有分，多写就多分。他的期末复习课很简略，甚至最后一节课还在讲新课，只有快下课的十几分钟简单说一下题型和重点，**一定要仔细听！一定要录音！** 因为我估计他是看着卷子给人们说的，~~他会泄露几道题~~。
6. 📍 **计算机网络** 同样是换老师，同样是毕业博士第一年教课，差别非常大。这个老师讲课差点意思，有点呆，而且讲课非常快，基本上半个多学期就讲完课了，然后有两节复习课，会给一个**模拟卷**，还会花一节课讲模拟卷。但是这个老师的考试非常好，仍然遵循老教师传统，往年题重复率很高，而且会有很多模拟卷的原题。
7. 📍 **人智导论** 平时不用听不用学，题型固定且简单，期末两天速成即可。
8. 📍 **企业实训/企业ai** 不用投入太大精力，基本都是3.7以上，稍微用点心就4.0。
9. 📍 **软件工程** 平时不用听，期末两天速成，把老师给的总结ppt从头到尾反复地看，因为是英文，想全背下来很费劲，而且有很多专业名词，所以只要来来回回多看几遍能有印象就行。
10. 📍 **软件体系结构** 这个老师好像也是第一年教课，不太确定。期末会把考试题直接给出，会给出两倍的题库，比如说考十道选择题那么他会给出二十道题库，最终会在题库中抽取各题型一半的题目进行考试，据说另一半用来缓考补考。**完全考原题！**而且题库这么少，属于是菩萨显灵了，这你背不下来属于是该挨嘴巴子了。
#### 5. E课 💦
两门推荐的E课，夯中之夯，能抢到属于是菩萨显灵了：

- **女性艺术家的艺术与情感**
- **认知脑科学-解读心智**
